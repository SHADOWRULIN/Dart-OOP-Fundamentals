# 🎯 Dart OOP Fundamentals: Midterm Project

[cite_start]This repository contains the implementation of core Object-Oriented Programming (OOP) concepts in Dart[cite: 1, 4]. [cite_start]I developed this project as a midterm assessment for the **Mobile App Development - 1** course at **Bano Qabil 2.0**[cite: 1, 2].

## 📝 Project Overview
[cite_start]The project consists of two primary challenges designed by **Instructor Muhammad Salman Bediya** to test proficiency in class structures, attribute management, and logic implementation in Dart[cite: 3, 5]:

### 1. BankAccount Management System
* [cite_start]**Attributes**: Account Number (int), Balance (double), Account Type (string), and Interest Rate (double)[cite: 8, 9, 11, 12].
* **Functionality**:
    * [cite_start]**Deposit**: Adds a specified amount to the balance[cite: 13].
    * [cite_start]**Withdraw**: Subtracts an amount from the balance, with validation to prevent overdrawing[cite: 14].
    * [cite_start]**Add Interest**: Calculates and adds interest to the balance based on the assigned rate[cite: 15].
    * [cite_start]**Display**: Prints a full summary of the account details[cite: 16].

### 2. Student Information System
* [cite_start]**Attributes**: Name (string), ID (string), and a dynamic list of courses (strings)[cite: 22, 23, 24].
* **Functionality**:
    * [cite_start]**Add Course**: Dynamically adds a new course to the student's record[cite: 25].
    * [cite_start]**Drop Course**: Removes an existing course from the student's list[cite: 25].
    * [cite_start]**Display Courses**: Prints a complete list of the student's current academic profile[cite: 26].

## 🛠️ Technical Skills Demonstrated
* [cite_start]**Encapsulation**: Using classes to bundle data and methods together[cite: 7, 20].
* [cite_start]**Logic Validation**: Implementing conditional checks to ensure data integrity, such as preventing withdrawals greater than the current balance[cite: 14].
* [cite_start]**Collection Management**: Manipulating dynamic lists within an object-oriented context[cite: 24, 25].
* [cite_start]**Object Instantiation**: Creating and managing multiple unique instances of the same class[cite: 17, 27].
